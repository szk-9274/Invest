# Core Beliefs (コア設計原則)

1. 再現性優先
- バックテストやスクリーニングは決定論的に動作すること。未来データ参照は厳禁。

2. ステージ分離
- Stage1/Stage2/Backtest の疎結合を維持。上流は下流のファイル（output/）のみを参照する。

3. TDD 優先
- 新しいロジックや変更はテストから書く（RED→GREEN→REFACTOR）。外部 API は必ずモックする。

4. 表示とロジックの分離
- frontend は表示のみ。重い計算・戦略は python 側で実行する。

5. 機密情報の扱い
- ハードコーディング禁止。環境変数を利用すること。
