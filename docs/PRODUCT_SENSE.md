# PRODUCT_SENSE — プロダクト価値・ユーザー重視の設計思想

ミッション
- ユーザーがデータに基づいて信頼できる意思決定を行えるよう、透明性・再現性・説明性を最優先にプロダクトを設計する。

ユーザーフォーカス
- 明示的な期待値：出力（バックテスト結果・スクリーニング結果）は『過去の条件下での再現』であり、将来の成績を保証するものではない旨を明確に表示する。
- 使いやすさ：主要な操作（バックテストの実行、結果のダウンロード、パラメータ保存）は最小のクリックで実行できること。

信頼性と透明性
- 再現性：同じ入力は常に同じ出力を返す（決定的処理）。未来データ参照（look-ahead）は厳禁。
- 証跡の提示：重要な計算やパラメータは UI 上で確認でき、必要に応じて CSV/ログとして出力できる。

評価指標（プロダクト視点）
- 信頼スコア（ユーザーが結果を信用できるか）
- 実行成功率（バックテストの失敗率）
- ランタイム（長い処理は進捗表示を必須とする）
- ユーザー保持（繰り返し利用）

プロダクト判断の基準
- 新機能は『ユーザーの主要課題をいくつ解決するか』で評価する。
- 結果の誤解を招く可能性がある変更（指標名称の変更、可視化の変更など）はプロダクト/デザイン/エンジニアでレビューする。
